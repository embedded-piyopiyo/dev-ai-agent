## 1. テスト概要

### 目的

本テストは以下を目的とする。

1. 各ツール（`read` / `write` / `edit` / `glob` / `grep` / `bash`）が単体で正しく動作すること
2. Agenticループが Anthropic Tool Use プロトコルに従って正しく動作すること
3. マルチターン会話の文脈が保持されること

### テスト種別

| 種別 | 説明 | API キー |
|------|------|---------|
| ユニットテスト（TC-U） | ツール実装関数を単体で検証。外部依存なし | 不要 |
| 統合テスト・モック（TC-I-01〜05） | `unittest.mock` で Claude API をモック化しループロジックを検証 | 不要 |
| 統合テスト・実API（TC-I-06〜07） | 実際の Claude API を呼び出してエンドツーエンドを検証 | 必要 |

### 4.2 統合テスト — Agentic Loop・モック（TC-I-01〜05）

**テストファイル**: `tests/test_agent.py` （`TestAgentLoopMocked` クラス）
**API キー**: 不要（`unittest.mock.MagicMock` で Claude API をモック化）

| TC ID | テスト名 | 目的 | 検証方法 | 合否基準 | 対応要件 |
|-------|---------|------|---------|---------|---------|
| TC-I-01 | end_turn でテキストを返す | `stop_reason=end_turn` のとき Claude のテキスト回答を返すこと | モックで `end_turn` を返す | 戻り値が期待テキストと一致し、API 呼び出しが1回 | REQ-07 |
| TC-I-02 | 正しいモデルが使用される | `claude-haiku-4-5-20251001` が API に渡されること | API 呼び出し引数を検証 | `call_args.kwargs["model"]` が正しいモデル名 | REQ-01 |
| TC-I-03 | tool_use 後ループ継続 | `stop_reason=tool_use` のあとループして `end_turn` で終了すること | 1回目 tool_use、2回目 end_turn を返すモック | 戻り値が最終テキスト、API 呼び出しが2回 | REQ-06, REQ-07 |
| TC-I-04 | messages が正しく蓄積される | messages リストに assistant/tool_result/assistant が順に追加されること | messages の長さと role を検証 | 長さ4、roles が user→assistant→user→assistant | REQ-03, REQ-06 |
| TC-I-05 | system プロンプトが送信される | API 呼び出しに system プロンプトが含まれること | API 呼び出し引数を検証 | `call_args.kwargs["system"]` が空でない | REQ-06 |

**テストファイル**: `tests/test_agent.py` （`TestAgentLoopReal` クラス）
**API キー**: 必要（`.env` の `ANTHROPIC_API_KEY`）

| TC ID | テスト名 | 目的 | 操作 | 合否基準 | 対応要件 |
|-------|---------|------|------|---------|---------|
| TC-I-06 | 実ツール呼び出し | glob ツールを実際に呼び出し結果を返答に含めること | `*.py` ファイルを検索するよう指示 | 返答が空でなく、`agent.py` か `main.py` が含まれる | REQ-04, REQ-07 |
| TC-I-07 | マルチターン文脈保持 | 前のターンの情報を次のターンで参照できること | ターン1でファイル名を伝え、ターン2で引き出す | ターン2の返答に `agent.py` が含まれる | REQ-03 |
