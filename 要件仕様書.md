## 1. 目的・背景

Anthropic SDK を直接使い、AIエージェントの仕組みを自ら実装することで理解を深める。
フレームワーク（pydantic-ai 等）に頼らず「Tool use プロトコル」「Agentic Loop」を自分の手で書くことが目的である。

---

## 2. 機能要件

| ID     | 要件                                                                 |
|--------|----------------------------------------------------------------------|
| REQ-01 | Claude Haiku 4.5（`claude-haiku-4-5-20251001`）API を使用すること    |
| REQ-02 | 対話型 CLI（REPL スタイル）で動作すること                            |
| REQ-03 | ユーザーとの会話コンテキスト（マルチターン）を保持すること           |
| REQ-04 | ファイル操作機能（読込・書込・編集・パターン検索・内容検索）を持つこと |
| REQ-05 | シェルコマンド実行機能（bash）を持つこと                             |
| REQ-06 | Anthropic Tool Use プロトコルを自前で実装すること（フレームワーク不使用） |
| REQ-07 | ツールが必要な限り自律的にループし、不要になったらユーザーに返答すること |

---

## 3. 非機能要件

| ID      | 区分       | 要件                                                              |
|---------|------------|-------------------------------------------------------------------|
| NREQ-01 | 言語       | Python 3.12 で実装すること                                        |
| NREQ-02 | ライブラリ | `anthropic`・`python-dotenv` のみ使用すること（最小限）           |
| NREQ-03 | 可読性     | 学習用途のため、コードにコメントで処理の意図を記述すること        |
| NREQ-04 | 拡張性     | 新しいツールを追加しやすい構造にすること                          |

---

## 4. 機能の詳細

### REQ-04 ファイル操作ツール

| ツール名 | 機能                                   |
|---------|----------------------------------------|
| `read`  | ファイルを読み込む（行範囲指定可能）   |
| `write` | ファイルを新規作成または上書き保存する |
| `edit`  | ファイル内の文字列を置換する（初回一致） |
| `glob`  | glob パターンでファイルパスを列挙する  |
| `grep`  | 正規表現でファイル内容を検索する       |

### REQ-05 Bash 実行ツール

| ツール名 | 機能                                          |
|---------|-----------------------------------------------|
| `bash`  | シェルコマンドを実行し stdout/stderr を返す   |

---

## 5. 検証基準

| REQ    | 確認方法                                                              |
|--------|-----------------------------------------------------------------------|
| REQ-01 | `agent.py` が `claude-haiku-4-5-20251001` モデルで API 呼び出しすること |
| REQ-02 | `python main.py` でプロンプトが表示され、入力待ちになること           |
| REQ-03 | 複数ターンの会話で前の文脈を踏まえた返答ができること                  |
| REQ-04 | 各ツール（read/write/edit/glob/grep）が正常に動作すること             |
| REQ-05 | `echo hello` 等のコマンドが実行されて結果が返ること                   |
| REQ-06 | `tool_use` / `tool_result` ブロックを自前コードで処理していること     |
| REQ-07 | 複数ツールを連続使用するタスクでループが正しく動作すること            |
